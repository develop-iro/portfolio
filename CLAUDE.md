## Commands

```bash
npm run dev        # Start dev server (http://localhost:4321)
npm run build      # Production build — static output to dist/
npm run preview    # Serve the dist/ build locally
npm run test       # Run tests once
npm run test:ui    # Interactive Vitest UI
npm run lint       # ESLint
npm run format     # Prettier (write)
```

To run a single test file:

```bash
npx vitest run src/test/SomeComponent.test.tsx
```

## Environment Setup

Copy `.env.example` to `.env` and fill in EmailJS credentials for the contact form to work:

```
PRIVATE_EMAILJS_SERVICE_ID=...
PRIVATE_EMAILJS_TEMPLATE_ID=...
PRIVATE_EMAILJS_PUBLIC_KEY=...
```

These are Astro `PUBLIC_*` variables — intentionally client-side. EmailJS public keys are designed to be exposed in the browser.

## Architecture

Static portfolio built with **Astro + React islands**. Astro generates static HTML; React is used only for interactive components (project filtering, theme toggle, hamburger menu, contact form/modal).

Follows **Feature-Sliced Design lite** — each domain is self-contained, shared primitives live in `shared/`, pages only orchestrate features.

```
src/
├── features/           # Self-contained domain modules
│   ├── hero/           # HeroSection.astro
│   ├── about/          # AboutSection.astro
│   ├── skills/         # data.ts + SkillsSection.astro
│   ├── projects/       # data.ts + ProjectsSection.astro + ProjectCard.astro + ProjectFilter.tsx
│   ├── experience/     # data.ts + ExperienceSection.astro
│   ├── resume/         # ResumePage.astro (renders experience + skills data)
│   └── contact/        # ContactSection.astro + ContactModal.tsx + ContactForm.tsx
├── shared/
│   ├── i18n/           # translations.ts — single source of truth for all UI strings
│   └── ui/             # Nav.astro, Footer.astro, ThemeToggle.tsx, HamburgerMenu.tsx,
│                       # CountUp.tsx, CustomCursor.astro
├── layouts/
│   └── BaseLayout.astro  — Root layout: SEO head, theme init, skip link, scroll bar, footer
├── pages/
│   ├── index.astro           — Language-detection redirect (no-JS fallback: meta refresh)
│   ├── en/index.astro        — English home
│   ├── en/resume.astro       — English resume page
│   ├── es/index.astro        — Spanish home
│   ├── es/resume.astro       — Spanish resume page
│   └── projects/[slug].astro — Static project detail page
└── test/
```

**Key rules:**

- Features import from `../../shared/` but never from other features directly.
- `shared/` contains no business logic — only generic UI primitives and i18n.
- Data lives co-located with its feature (`features/*/data.ts`). To add content, edit the relevant `data.ts`.
- React components use `client:load` or `client:visible` directives. Never add `client:*` to static-only components.
- Path alias `@/*` maps to `src/*`.

**i18n:** Routes duplicated under `src/pages/en/` and `src/pages/es/`. All UI strings live in `src/shared/i18n/translations.ts`. To add a key, add it to both `en` and `es` objects — the `UIKey` type updates automatically.

**Styling:** Tailwind CSS with class-based dark mode (`dark:` prefix). Use `motion-safe:` / `motion-reduce:` variants for animations. Supports 320px–2560px viewports.

**Testing:** Vitest + jsdom + React Testing Library. Property-based tests use `fast-check`. Test files live in `src/test/`.

## SEO

`BaseLayout.astro` accepts these optional props beyond `title`, `description`, and `lang`:

| Prop | Type | Purpose |
|------|------|---------|
| `ogImage` | `string` | Absolute path to OG image. Defaults to `/images/profile.jpeg`. |
| `alternates` | `{ hreflang: string; href: string }[]` | Rendered as `<link rel="alternate" hreflang>` tags. |

Every page already sets `alternates`. The sitemap is auto-generated by `@astrojs/sitemap` on build.

**Site URL:** `https://ivan-romero.dev` — configured in `astro.config.mjs`. Update here if the domain changes; also update the hardcoded `alternates` URLs in each page and `public/robots.txt`.

## Deployment

Deployed on **Vercel**. Config lives in `vercel.json` at the root:

- HTTP security headers (CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy) applied to all routes.
- No special rewrites needed — `output: 'static'` generates plain HTML.

Vercel auto-provisions TLS for the custom domain. Set the three `PUBLIC_EMAILJS_*` environment variables in **Vercel → Settings → Environment Variables** for Production (and Preview if needed).
